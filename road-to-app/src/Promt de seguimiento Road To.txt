🚗 ROAD TO APP - CONTEXTO PARA CONTINUAR (EXPLICADO PARA PRINCIPIANTES)

=== ¿QUÉ TENGO HASTA AHORA? ===
- Una app que funciona en teléfonos Android
- Los conductores pueden ver un mapa y su ubicación en tiempo real
- PERO: Si el conductor abre WhatsApp o Spotify, la app deja de rastrearlo
- Necesito que siga funcionando aunque el conductor use otras apps

=== ¿QUÉ NECESITO HACER? ===

PRIORIDAD #1: QUE LA APP NO SE "DUERMA" (Background Tracking)
Problema actual: El GPS se apaga cuando la app no está en pantalla
Solución: Crear un "servicio persistente" que Android no pueda matar fácilmente

PASOS QUE NECESITO (explicados mañana):
1. Abrir Android Studio (el programa para editar la parte de Android)
2. Crear un archivo nuevo en Kotlin (lenguaje de programación de Android)
3. Copiar código que me darán para mantener el GPS siempre activo
4. Modificar 2-3 archivos para conectar todo
5. Probar en un teléfono real

PRIORIDAD #2: ZONAS ESPECIALES (Geofencing)
Problema actual: El admin no sabe cuándo un conductor llega a un punto importante
Solución: Dibujar círculos en el mapa. Cuando el conductor entra, avisar al admin

PASOS QUE NECESITO:
1. Crear tabla nueva en Supabase (mi base de datos) llamada "zonas"
2. Agregar en el panel del admin: un mapa para dibujar zonas
3. Cuando el conductor entre a una zona: enviar notificación al admin
4. El admin ve una lista: "Juan Pérez quiere entrar a Zona Norte"
5. El admin aprieta "Aceptar" y el conductor entra a la fila de espera
